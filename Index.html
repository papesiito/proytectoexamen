Pasos del flujo de trabajo:

  1. Creación del proyecto HTML: Proyecto HTML generado (index.html).
  2. Inicialización del repositorio: git init
  3. Primer commit: git add . → git commit -m "Initial commit"
  4. Conexión con GitHub: git remote add origin [URL del repositorio]
  5. Subida al repositorio: git push -u origin main
  6. Fork por otro usuario: El otro usuario hace un fork desde GitHub.
  7. Clonación del fork: git clone [URL del fork]
  8. Creación de una rama: git branch feature-update → git checkout feature-update
  9. Modificaciones en el código: Se realizan cambios en el proyecto HTML.
  10. Commit de los cambios: git add . → git commit -m "Feature update"
  11. Push de la rama: git push origin feature-update
  12. Apertura de Pull Request: Desde GitHub, se abre una Pull Request para revisión.
  13. Revisión y cierre de Pull Request: Se revisan los cambios y se acepta (merge) en la rama principal.

¡Flujo completado exitosamente!
